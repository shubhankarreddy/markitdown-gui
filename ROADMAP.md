# Roadmap

This roadmap prioritizes practical usefulness for people converting documents into AI-ready Markdown.

## Near Term

- Add official screenshots and a short demo GIF to the README.
- Add a license after the maintainer chooses one.
- Add sample fixture documents for manual release testing.
- Improve empty states in the preview workspace.
- Add drag reorder or batch grouping for queue items.

## Packaging

- Add an installer build once Inno Setup 6 is available.
- Add version metadata to the Windows executable.
- Publish GitHub Releases with the standalone EXE and installer.
- Consider code signing if the app gains public users.

## Conversion Workflow

- Add per-item output filename editing.
- Add optional overwrite or skip behavior for existing Markdown files.
- Add export log for failed conversions.
- Add clearer support matrix for file types handled by MarkItDown.

## Later

- Bring the WPF native client to feature parity or clearly retire it.
- Add persistent queue sessions.
- Add template options for frontmatter and Markdown cleanup.
- Add automated smoke tests for queue and save behavior.
